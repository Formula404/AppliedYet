# 投了吗（Applied Yet?）

> 从投递到入职，把每一步都记清楚。

“投了吗”是一款面向 Windows 的本地优先个人求职工作台。它以一次具体的岗位投递为中心，把公司、岗位、简历、招聘邮件、面试准备、模拟面试、真实面试复盘与 Offer 关联在同一条可追溯的求职流程中。

当前仓库已完成产品基础骨架、核心投递数据闭环和外部服务配置底座，正在逐步接入 AI、ASR、邮箱与本地文件系统能力。

## 当前已实现

- 桌面工作台：固定侧边栏、顶部搜索、快捷操作、通知与邮件同步反馈；
- 首页：月历、KPI、投递漏斗、今日待办、最近邮件、本周进展与重点岗位；
- 我的投递：岗位看板/列表切换、搜索筛选、新增投递表单；
- 招聘邮件：邮件列表、识别结果、匹配依据、置信度与确认更新流程；
- 面试准备：岗位理解、简历匹配、准备清单与预测问题；
- 模拟面试：岗位、类型、难度、题量、语音/文字方式等配置；
- 面试复盘：本地文件导入入口、待复盘记录与持续改善项；
- 个人题库：问题库、故事库与薄弱项入口；
- Offer：多 Offer 维度对比和用户自定义权重的界面基础；
- 数据分析：投递转化、岗位分布和可执行建议；
- 设置：持久化 AI/ASR 服务参数、逐项控制外发数据范围，并通过 Windows 凭据管理器安全保存 API Key；
- 全局交互：`Ctrl + K` 搜索、`Ctrl + N` 新增投递、减少动态效果适配；
- Tauri 2 桌面入口：Windows 窗口尺寸、最小尺寸和基础权限配置。
- SQLite 本地数据库：版本化迁移、完整性检查，以及公司/岗位/投递/事件/任务基础表；
- 投递数据闭环：桌面端支持从数据库查询、新增投递，并持久化看板阶段变更。
- 岗位/投递详情：支持编辑公司、岗位、投递阶段、下一步行动和 JD；
- 任务与事件时间线：支持新增/完成岗位任务、手动补记事件，并自动记录资料、阶段和任务变化。
- 首页真实聚合：KPI、动态月历和今日待办从本地投递与任务数据实时生成。
- 管理闭环：支持任务编辑/删除、投递归档/恢复，以及带冲突校验的阶段事件撤销；
- Windows 任务提醒：根据任务提醒时间轮询并发送系统通知，数据库记录发送状态避免重复通知。
- AI Provider：支持连接测试、Responses API 结构化输出、解析失败重试、备用模型，以及按投递记录来源、尝试次数和耗时；
- AI 面试准备：根据岗位 JD 与投递上下文生成并持久化重点准备、预测问题和行动计划；
- 文档与语音入口：本地解析 PDF、DOCX、TXT、Markdown，并通过可配置 ASR Provider 转写音频。
- 个人档案：支持多份简历上传解析、默认简历、软删除，以及个人信息、教育背景、实习经历、项目经历、专业技能、学术成果、技能证书逐栏编辑；
- 简历工作流：支持空白新建、复制、派生版本、适用方向与备注、归档恢复、投递绑定、历史版本防覆盖、版本转化统计和原文核对；
- 简历驱动 AI：岗位准备会结合关联简历生成匹配分析，模拟面试可基于该版本生成带来源依据的真实简历深挖题；
- AI 厂商预设：支持 OpenAI、Claude、Gemini、DeepSeek、通义千问、智谱 GLM、Moonshot、硅基流动和自定义 OpenAI 兼容服务，并自动填充协议、接口地址和模型。

桌面端的投递、详情、任务、事件时间线、首页聚合、Provider 设置和招聘邮件流程已使用真实本地数据库；招聘邮件支持 IMAP 增量检查、本地识别、投递匹配、人工确认及阶段更新。浏览器开发模式不连接桌面 SQLite、系统凭据或真实邮箱，自动使用一套与真实数据完全隔离的完整演示数据，适合产品演示、截图与宣传录屏；刷新页面即可恢复预置状态。

## 技术栈

- 桌面框架：Tauri 2 / Rust
- 前端：React 18、TypeScript、Vite
- 路由：React Router
- 图标：Lucide React
- 样式：原生 CSS Design Tokens，未依赖重量级 UI 框架
- 规划中的本地能力：SQLite + FTS5、Python Sidecar、FFmpeg

项目结构遵循产品技术文档的分层建议：

```text
.
├── apps/
│   └── desktop/
│       ├── src/
│       │   ├── app/          # 路由入口
│       │   ├── components/   # 应用框架与通用组件
│       │   ├── data/         # 演示数据（后续由服务层替换）
│       │   ├── pages/        # 业务页面
│       │   ├── styles/       # 全局设计系统与响应式样式
│       │   └── types/        # 共享前端类型
│       └── src-tauri/        # Tauri / Rust 桌面入口
├── docs/                     # 产品、技术、UI 文档与概念图
├── package.json              # 根目录快捷命令
└── README.md
```

## 本地运行

环境要求：

- Node.js 20 或更高版本；
- npm 10 或更高版本；
- 如需运行桌面壳：Rust stable 与 Tauri 2 的 Windows 系统依赖。

安装依赖：

```bash
cd apps/desktop
npm install
```

返回仓库根目录启动浏览器开发模式：

```bash
npm run dev
```

默认访问地址为 `http://localhost:1421`。在普通浏览器中打开时，顶部会显示“演示数据”标识；由 Tauri 窗口加载同一地址时则会使用 Tauri 桥接并连接桌面应用的本地数据库。

运行 Tauri 桌面模式：

```bash
npm run tauri dev
```

执行类型检查与生产构建：

```bash
npm run typecheck
npm run build
```

## 设计原则

实现遵循 `docs/投了吗_UI设计规范.md`：

- 日历是首页主要工作区，而不是装饰；
- 蓝色用于主要操作，绿/橙/紫/青/红严格表达业务状态；
- 首页保持高信息密度，但通过卡片、间距和字号层级控制阅读节奏；
- 自动匹配展示来源、依据、置信度，并保留用户确认入口；
- AI 不占据首页中心，也不替用户做 Offer 决策；
- 最小窗口尺寸为 1024 × 720，并适配 1440 × 900 的主要设计基准。

3. 增加自动备份、恢复与数据导出；
4. 实现 OAuth/IMAP 增量同步、规则优先的邮件分类与可撤销更新；
5. 增加 Python Sidecar，接入文档解析、ASR Provider 与任务进度；
6. 完成后台任务、错误重试、日志脱敏与安装包验证。

## 隐私说明

项目坚持本地优先：业务数据、简历、邮件索引和面试文件默认保存在用户设备上。仅在用户主动启用 AI、语音识别或邮箱能力时连接第三方服务；正式实现中应在每次外部调用前明确数据类型、用途与发送范围。
