# 投了吗｜UI 设计规范

> 适用于“投了吗（Applied Yet?）”Windows 桌面应用的界面设计、前端实现与后续迭代。

---

## 1. 文档信息

- 产品名称：投了吗
- 英文名称：Applied Yet?
- 文档类型：UI 设计规范
- 版本：1.0
- 日期：2026-07-13
- 适用平台：Windows 10 / Windows 11 桌面端
- 设计目标：清晰、克制、高效、可信、可长期使用

---

# 2. 设计定位

“投了吗”是一款以岗位投递为核心的个人求职工作台。

界面不应表现为传统招聘网站，也不应表现为普通 AI 对话工具。整体视觉需要同时体现：

- 日程管理工具的清晰；
- 数据中台的专业；
- 个人工作台的温和；
- 求职过程的紧迫感；
- 本地工具的可靠与克制。

核心视觉关键词：

```text
轻量
克制
清晰
专业
可信
有节奏
低打扰
高信息密度
```

# 3. 核心设计原则

## 3.1 日历优先

首页以日历为主要操作中心，所有关键日期都应在日历中清晰呈现：

- 测评截止；
- 笔试；
- 面试；
- HR 沟通；
- Offer 截止；
- 面试复盘；
- 自定义任务。

日历不是装饰，而是首页的主要工作区。

## 3.2 数据辅助决策

数据看板用于帮助用户理解求职状态，不追求复杂的商业 BI 效果。

数据应回答：

- 我投了多少？
- 现在有多少岗位在进行？
- 哪些岗位需要马上处理？
- 哪个环节转化率最低？
- 最近一周是否有进展？
- 哪些岗位最值得关注？

## 3.3 操作优先于展示

首页上的每个主要模块都应支持直接操作，例如：

- 点击日历事件进入岗位详情；
- 点击待办直接完成或延期；
- 点击邮件进入匹配确认；
- 点击岗位卡进入完整流程；
- 点击数据指标进入筛选后的列表。

不允许出现只有视觉效果、没有实际作用的卡片。

## 3.4 颜色表达状态

颜色只用于表达流程和优先级，不作为纯装饰。

同一状态在全产品中必须保持一致：

- 测评与笔试使用绿色；
- 面试使用蓝色；
- HR 沟通使用橙色；
- Offer 相关使用青绿色；
- 复盘与准备任务使用紫色；
- 拒绝、异常和高风险使用红色；
- 无反馈和暂停使用灰色。

## 3.5 高信息密度但不拥挤

首页允许展示较多信息，但必须通过以下方式保持清晰：

- 明确分区；
- 足够留白；
- 一致的卡片结构；
- 统一的字号层级；
- 颜色数量受控；
- 重要信息靠近视觉中心；
- 次要信息使用弱化色。

## 3.6 自动化必须可解释

邮件自动匹配、流程自动更新和 AI 分析结果必须展示：

- 来源；
- 匹配依据；
- 置信度；
- 更新时间；
- 是否可撤销。

# 4. 页面总体结构

## 4.1 应用窗口

建议最小窗口尺寸：

```text
1280 × 800 px
```

推荐设计基准尺寸：

```text
1440 × 900 px
```

高保真稿建议使用：

```text
1600 × 900 px
```

不建议将窗口设计为移动端式单列布局。

## 4.2 页面骨架

```text
┌──────────────────────────────────────────────┐
│ 顶部工具栏                                  │
├───────────┬──────────────────────────────────┤
│ 左侧导航栏 │ 主内容区                         │
│           │                                  │
│           │                                  │
│           │                                  │
└───────────┴──────────────────────────────────┘
```

## 4.3 左侧导航栏

推荐宽度：

```text
200–220 px
```

折叠状态宽度：

```text
64–72 px
```

导航栏固定，不随主内容滚动。

## 4.4 顶部工具栏

推荐高度：

```text
64–72 px
```

包含：

- 全局搜索；
- 当前日期；
- 邮件同步；
- 通知；
- 用户入口；
- 窗口控制。

## 4.5 主内容区

主内容区采用 12 栏网格。

推荐布局：

```text
日历：7 栏
数据区：5 栏
```

下方模块可采用：

```text
最近邮件：3 栏
本周进展：4 栏
重点岗位：5 栏
```

# 5. 栅格与间距

## 5.1 基础间距单位

使用 4 px 为基础单位。

推荐间距：

| 用途 | 数值 |
|---|---:|
| 极小间距 | 4 px |
| 图标与文字 | 8 px |
| 紧凑内容 | 12 px |
| 常规组件内边距 | 16 px |
| 卡片间距 | 20 px |
| 页面分区间距 | 24 px |
| 大区块间距 | 32 px |
| 页面外边距 | 24–32 px |

## 5.2 页面边距

主内容区左右边距：

```text
24 px
```

大屏可增加至：

```text
32 px
```

## 5.3 卡片间距

同级卡片之间：

```text
16–20 px
```

卡片内部：

```text
16–24 px
```

## 5.4 对齐规则

- 卡片标题左对齐；
- 数据值左对齐；
- 时间标签右对齐；
- 图表坐标轴保持一致；
- 多张 KPI 卡片底部对齐；
- 列表图标、标题和状态标签保持基线对齐。

## 6.4 颜色使用限制

- 单个页面主要强调色不超过 5 种；
- 单个卡片内最多使用 2 种强调色；
- 不使用高饱和渐变作为大面积背景；
- 不用颜色作为唯一识别方式，需配合文字或图标；
- 红色只用于风险、拒绝、失败和紧急事项；
- 绿色不用于普通成功装饰，只表示完成、通过或 Offer。

# 7. 字体规范

## 7.1 推荐字体

Windows 中文环境：

```text
Microsoft YaHei
Segoe UI
PingFang SC
Noto Sans SC
sans-serif
```

推荐 CSS：

```css
font-family:
  "Segoe UI",
  "Microsoft YaHei",
  "PingFang SC",
  "Noto Sans SC",
  sans-serif;
```

## 7.2 字号层级

| 层级 | 字号 | 字重 | 用途 |
|---|---:|---:|---|
| 页面主标题 | 24 px | 600 | 页面名称 |
| 区块标题 | 18 px | 600 | 日历、今日待办 |
| 卡片标题 | 15–16 px | 600 | 最近邮件、重点岗位 |
| 关键数据 | 28–32 px | 600 | 累计投递、Offer |
| 正文 | 14 px | 400 | 列表、说明 |
| 辅助文字 | 12–13 px | 400 | 时间、来源、状态 |
| 微型标签 | 11–12 px | 500 | 状态标签 |

## 7.3 字重

建议只使用：

```text
400 Regular
500 Medium
600 Semibold
```

不建议使用 700 以上的粗体，避免界面显得沉重。

## 7.4 行高

- 标题：1.3；
- 正文：1.5；
- 列表：1.4；
- 长文本：1.6。

# 8. 圆角与阴影

## 8.1 圆角

```css
--radius-small: 6px;
--radius-medium: 10px;
--radius-large: 14px;
--radius-card: 16px;
--radius-pill: 999px;
```

使用建议：

- 状态标签：6–8 px；
- 输入框：8–10 px；
- 普通按钮：8–10 px；
- 卡片：14–16 px；
- 胶囊按钮：999 px。

## 8.2 阴影

卡片默认：

```css
box-shadow:
  0 1px 2px rgba(16, 24, 40, 0.04),
  0 4px 12px rgba(16, 24, 40, 0.04);
```

浮层：

```css
box-shadow:
  0 8px 24px rgba(16, 24, 40, 0.12);
```

不要使用过重阴影。卡片主要依靠边框和背景层级区分。

# 9. 左侧导航规范

## 9.1 导航结构

固定顺序：

1. 首页
2. 我的投递
3. 招聘邮件
4. 面试准备
5. 模拟面试
6. 面试复盘
7. 个人题库
8. Offer
9. 数据分析
10. 设置

## 9.2 导航项尺寸

推荐：

```text
高度：44–48 px
左右内边距：14–16 px
图标：18–20 px
文字：14 px
```

## 9.3 导航状态

### 默认

- 图标：中灰；
- 文字：深灰；
- 背景：透明。

### 悬停

- 背景：浅灰蓝；
- 图标和文字：主色。

### 选中

- 背景：`#EAF1FF`；
- 图标：主蓝；
- 文字：主蓝；
- 字重：600。

## 9.4 导航角标

适用于：

- 待确认邮件；
- 未完成复盘；
- 待处理任务。

角标应使用小圆点或数字，不超过两位数。

# 12. 日历设计规范

## 12.1 日历模式

首页默认显示月视图。

后续可支持：

- 周视图；
- 日程列表；
- 时间轴视图。

## 12.2 日期单元格

每个日期单元格包含：

- 日期数字；
- 事件标签；
- 超出事件数量；
- 状态提示。

单元格最小高度：

```text
72–88 px
```

## 12.3 事件标签

事件标签结构：

```text
状态图标 + 简短事件 + 公司简称
```

示例：

```text
一面 · 腾讯
测评截止 · Shopee
Offer 截止
复盘任务 2
```

## 12.4 事件颜色

| 类型 | 颜色 |
|---|---|
| 测评 / 笔试 | 绿色 |
| 面试 | 蓝色 |
| HR 沟通 | 橙色 |
| Offer | 青绿色 |
| 复盘 | 紫色 |
| 异常 / 逾期 | 红色 |
| 普通任务 | 灰蓝色 |

## 12.5 当日样式

- 日期数字使用主色圆形背景；
- 单元格增加浅蓝边框；
- 不使用大面积高饱和填充。

## 12.6 选中日期

选中日期展示：

- 轻微主色描边；
- 当前日期圆点；
- 右侧或浮层显示当天详情。

## 12.7 事件过多

同一天最多展示 3 条事件。

超出后显示：

```text
+3 项
```

点击后打开当天详情。

## 12.8 拖拽

允许将用户手动任务拖动到其他日期。

邮件自动生成的关键时间默认不允许直接拖动，需进入详情修改。

# 13. KPI 卡片规范

## 13.1 KPI 内容

首页建议固定展示：

- 累计投递；
- 进行中；
- 待测评；
- 待面试；
- Offer；
- 拒绝率或整体转化率。

## 13.2 卡片结构

```text
指标名称
关键数值
占比或环比
小型状态图标
```

## 13.3 数值样式

- 主值：28–32 px；
- 环比：12 px；
- 增长使用绿色；
- 风险增长使用红色；
- 不直接使用红绿表达好坏时，应附文字。

## 13.4 空状态

没有数据时显示：

```text
暂无数据
完成第一次投递后，这里会出现统计结果
```

不要显示大量 0 和空图表。

# 14. 今日待办规范

## 14.1 列表项内容

每项包括：

- 优先级圆点；
- 任务标题；
- 关联岗位；
- 时间；
- 状态；
- 快捷操作。

## 14.2 优先级

- 高：红色；
- 中：橙色；
- 普通：蓝色；
- 低：灰色。

## 14.3 快捷操作

悬停显示：

- 完成；
- 延期；
- 打开岗位；
- 编辑。

## 14.4 逾期

逾期任务：

- 时间显示红色；
- 标签显示“已逾期”；
- 不使用整行红色背景。

默认：

投递

## 15.2 展示内容

# 16. 最近招聘邮件规范

包含：

- 公司与岗位；
- 邮件类型；
- 摘要；
- 时间；
- 匹配状态。

## 16.2 匹配状态

- 已自动匹配；
- 待确认；

# 17. 重点岗位规范

## 17.1 卡片内容

每个岗位包含：

- 公司；
- 岗位；
- 地点；
- 当前阶段；
- 下一步；
- 时间；
- 流程进度。

## 17.2 流程进度

推荐：

```text
投递 → 测评 → 一面 → 二面 → Offer
```

当前阶段使用主色，已完成阶段使用浅色实心，未完成使用灰色。

## 17.3 风险提示

以下情况显示轻量提示：

- 三天未更新；
- 测评即将截止；
- 面试未准备；
- 邮件待确认；
- Offer 即将到期。

# 18. 图表规范

## 18.1 图表用途

只展示用户能理解并能采取行动的数据。

不展示：

- 复杂 3D 图表；
- 过多渐变；
- 装饰性雷达图；
- 无解释的评分仪表盘。

## 18.2 折线图

用于：

- 投递趋势；
- 面试趋势；
- 每周进展；
- 模拟面试表现。

要求：

- 最多 3 条线；
- 主线使用主蓝；
- 辅助线使用绿色或灰色；
- 鼠标悬停显示详细值。

## 18.3 柱状图

用于：

- 行业分布；
- 岗位方向；
- 投递渠道；
- 简历版本对比。

## 18.4 环形图

用于：

- 简历通过率；
- 面试通过率；

# 35. 设计验收清单

## 35.1 一致性

- 状态颜色是否统一；
- 圆角是否统一；
- 字号层级是否统一；
- 图标是否同一风格；
- 按钮文案是否使用动词；
- 时间格式是否一致。

## 35.2 可用性

- 用户是否能快速找到今日任务；
- 用户是否能看清下一场面试；
- 邮件待确认是否明显；
- 重点岗位是否容易进入；
- 日历事件是否可点击；
- 数据卡片是否能跳转。

## 35.3 信息密度

- 是否一屏内看到核心信息；
- 是否存在重复数据；
- 是否有卡片只用于装饰；
- 是否有过多标签；
- 是否出现不必要的留白。

## 35.4 状态反馈

- 加载是否有反馈；
- 失败是否说明原因；
- 自动更新是否可撤销；
- 危险操作是否二次确认；
- 后台任务是否可查看进度。

## 35.5 隐私

- 是否清楚标记外部调用；
- 是否避免展示完整敏感内容；
- 是否对邮箱和简历信息进行适当隐藏；
- 是否存在无意上传用户数据的入口。

# 36. 视觉总结

“投了吗”的 UI 应当像一个成熟的个人工作台，而不是招聘平台，也不是 AI 工具集合。

首页的核心体验是：

```text
看日历，知道今天做什么；
看数据，知道求职走到哪里；
看邮件，知道流程发生了什么；
看岗位，知道下一步该准备什么。
```

视觉上应保持：

- 白色和浅灰为主；
- 蓝色作为品牌强调；
- 状态色严格受控；
- 卡片清晰但不过度立体；
- 信息多但不拥挤；
- 自动化强但不打扰；
- AI 存在于功能中，而不是存在于视觉口号中。
